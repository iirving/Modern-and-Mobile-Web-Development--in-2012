CSS : Pre History

Prior to CSS (Cascading Style Sheets), nearly all of the presentational attributes of HTML documents were contained within the HTML markup.

All font, colors, font colors, background styles, element alignments, borders and sizes had to be explicitly described, repeatedly, within the HTML.

It was a MESS!

CSS : Getting Presentation out of Content.

By 1999 all of that out of the HTML and into embedded CSS or in a linked CSS files

so the markup no longer had a thousand references to FONT and COLOR.

which was a big improvement, more maintainable, smaller, easier to debug.

CSS : Table Driven Layout

But, there were a raft of problems of using Tables for layout structure:

  • a rats net of nested tables
  • the page didn't load until the whole table was rendered.
  • hard to modify the content or structure

there was a solution in hand,

(even if it took many years for us to integrate it into our development practices)

CSS : and along came frameworks

The other thing that allowed for a rapid embrace of Grid based layouts was the emergence of CSS Frameworks

They allowed for easier, more standards-compliant styling of web pages using best practices

  • A easy to use grid
  • A Sensible default typography and styling baseline
  • A browser CSS reset.
  • A stylesheet for printing

CSS : and along came frameworks

and the biggest difference between Frameworks being the size of the grid (920px, 1024px 1140px were commonly used ) and the number of columns (4 or 12 or 16 ).

and there are even grid generators:

plug in your total width, and number of max number of columns and BINGO!

U Haz Layout

They all help you development the layout and style of your sites with a foundation of best practices, quickly.

The BluePrint http://blueprintcss.org/ come out in 2007 and I started using it right away

But these were still Static grids!

CSS : Elastic Design

Now there had been some crazy people as far back as 2001 and 2002 who talked about "elastic design"

Like Dug Falby in A List Apart : Flexible Layouts , and Eric Costello

But they were ignored, because:

  • everyone had larger and larger laptop screens (17") or desktop monitors (+23")
  • we where still thinking static or even print layouts
  • they seemed like a lot of work to implement

So they were largely ignored

CSS : CSS Preprocessors

(extends and is interpreted into CSS.) It adds the stuff that should have been in CSS in the first place. things like variables, mixins, and functions, math! Write comment that don't end up in your production code It will make your CSS more maintainable by making it DRY, as in Don't Repeat Yourself. It will let you write better CSS (with better cross browser support) the production CSS output will be highly compressed and gzipped.

Mixin's : used to share comman patterns of properties, which can then be reused. 
	 
@mixin my-style {
		background: red;   color: white;   float: left;   display: block;   }		     

.box-1 {    @include my-style; 			  }

.box-2 {
	@inculde my-style;
	font-size: 16px;
}

=> 
.box-1 { background: red; color: white; float: left; display: block; }    
.box-2 { background: red; color: white; float: left; display: block; font-size: 16px; }
